Week 10 Notes

MDN: Validating Forms

Front end validate to make it user friendly. Back-end validate to maintain data integrity and security.

Built-in HTML5 validation performs better and is fairly simple to implement. JS validation more customizable.

HTML5- required, min/max, type, pattern

CSS- :valid styles valid form items. :invalid styles invalid items

:required targets required elements

pattern uses regular expressions to limit input

JS validation- most browsers use the Constraint validation API. Adds properties to form elements to handle common validation needs

novalidate disables browser's built in validataion

MDN: Using Fetch

fetch doesnt send cross origin requests without setting credentials in the init option

fetch has a lot of request options. Used to set method of the request, cors info, tracking, response body format

cors error returns a TypeError, successful fetch has Response.ok = true

headers pass mapped name-value pairs in the request. The guard property sets the rules of what limits there are with respect to the header object

response bodys can contain multiple types of data

fetch feature detection, if(window.fetch){}
