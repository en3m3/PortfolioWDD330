Novice to Ninja: Chapter 9 The Window Object

window is a global object that represents the browser window which contains the web page(web document)

Browser Object Window(BOM) good acronym. Seems familiar. Holds properties and methods for the browser and screen

BOM only significant in a browser windows. Other environments wont have the global.

there are many methods of windows that we have used without recognizing they were methods. (parseInt, isNaN, alert, confirm, prompt)

browser information is congainer in the Navigator object. navigator.userAgent returns browser and operating system being used

window.location returns the current url. location.href changes the url to a new one(redirects page) location.protocol returns the protocol (http:, https:,ftp:, and so on. location.hostname returns domain. location.port returns the port number(sometimes empty) location.pathname returns everything after the domain. location.search returns the url parameter string

window.location.hash returns any string following a # in the url. location.origin returns protocol and domain that the page originated from

window.location.reload() reloads the page. reload(true) forces uncached reload. location.assign() loads another resource from another location location.replace() does the same without keeping session data. location.toString() returns url as a string

window.history. history.go(int) goes forward or backwards the amount that is passed to it. history.forward and history.back goes forward or backwards one

window.open() opens a new window. parameters open(url, window title, attributes)

window.close() lets you close a window, like a popup

window.moveTo() moves the window to the coordinates given

window.resizeTo() resizes to the width and height passed to it.

window.screen holds screen information. screen.height, screen.width, screen.availWidth, screen.availHeight, screen.colorDepth

window.document. The document property contains the items on the page. document.write() writed to the document replacing anything that was previously contained

cookies sore key value pairs separated by semicolons. Persist past page reloading

create using document.cookie = 'key=value'; new additions are appended, doesnt override values unless the key is the same

to read cookies use split with ; as the delimiter

setting an expires value or max-aged allows the cookies to exist past the session

browsers handle expiration differently so you shouldn't rely on it for functionality.

secure cookies force https

to delete cookies set the expiration to a time in the past. Session cookies are erased when the window is closed

setTimeout runs a function after a certain delay, setInterval runs a function repeatedly after a set delay. clearInterval stops the timer/ends the interval/timeout

window.requestAnimationFrame() similar to setInterval except it uses the browsers animation capabilities. cancelAnimationFrame ends it

Most of the time CSS animations are better to use

Novice to Ninja: Chapter 14 HTML5 APIs

data attributes. begin with data-. May have a balue, but it can also be bool

data attributes are stored in an element's dataset property. When referencing then in the property the hyphens and removed and the name is converted to camel-case

getAttribute() can also get dataset values

web storage. The window object has a localStorage and sessionStorage property. Both are stored as key value pairs.

getItem and setItem can be used with the storages, as well as dot notation

delete and removeItem can remove an entry. clear() removes all entries.

when a value is saved to one of the storages it triggers a storage event. The properties the callback has are key, newValue, oldValue, and storageArea

Geolocation API is used to obtin the devices location. navigator.geolocation.getCurrentLocation() returns a position object that has a coords property that holds latitude and longitude among other things

watchPosition() calls a callback any time the location changes. clearWatch() removes it

Web Workers allow processes to run in the background. You use Worker() to create one.

stop wowrker with terminate() or close()

websockets allow two way communication. Push messaging.

Notification API. Need permission before using. requestPermission() method to get permission.

media api. Audio and video tags

canvas api

shim and polyfill help allow use of the newest API abilities without breaking the site for people with browsers that do not support them.
