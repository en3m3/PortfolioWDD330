Week 3 Notes - Novice to Ninja

Object Methods article: this

This used in an object's method refers to the object

Arrow functions do not have their own this so if used it will refer to the this of the outer normal function's this. Which is a useful trait of arrow functions that has many uses.

If you set up a method that returns this in an object it will return the object, this does not work as a property.

Object literals wrap properties and methods

Naming conventions camelcase thisObject and snake this_object

Looping through an object in a for loop you can display all the properties and functions as well as their values

Adding/changing/deleting new properties is the same as adding a new key-value pair.

Chapter 6 - Document Object Model

ways to get DOM objects: getElementById , getElementsByClassName , getElementsByTagName , querySelector and querySelectorAll

query selector is good for very specific targeting. Returns a non-live node/nodelist

queryselector() returns first occurence only, queryselectorall() returns all matching DOM elements

childrenNodes returns all children nodes, including text nodes of the element children only returns the children elements ignoring text nodes

firstChild on a node list returns first child node lastChild returns last

parentNode property returns the parent

nextSibling and previousSibling returns the adjacent nodes either before or after

nodeValue returns the text if used on a text node

className with replace all classes on a node classList you can add and remove specific classes

createElement creates a new node of the type you sent as a parameter

appendChild adds the node as a child of the element

textContent doesn't allow html, innerHTML does. if you aren't using html textContent is less resource intensive

getElementBy* methods return a live node which means if changed it will change the element in the DOM.

style property changes css

Chapter 7 - Events

inline event handlers are attributes in html tags such as onclick

inline breaks the concept of keeping html and js separate

addEventListener is prefered way now

event parameter is automatically sent from a event listener which contains information about the event, such as scrollbar offset after an onscroll event is triggered

mouse and touch events need to both be considered with current technology

if you will be removing an eventListener to not use an anonymous function when you add it, you will need to reference the function to remove it.

Event handling, capturing/bubbling/delegation

Capturing is useful when you have outside code you want to run before handling the event

Bubbling resolves from the element triggering the event up to the parents

Capturing propagate downwards from the parents to the children

You can use bubbling and capturing together if you add more listeners

Using stopPropagation() can prevent events from triggering much higher up in the bubbling to not fire which can cause problems if you aren't careful.
