Chapter 2 - Programming Basics

Primative Data Types: number, boolean, string, symbol, undefined, null

Non-primatives are objects

typeof on a variable returns data type

primatives are dynamic. They can adjust types automatically.

Sometimes the types being dynamic can cause undesired results. It's best to treat them strict to prevent issues.

There are many ways to convert variables to different data types but some are more hacky and should be avoided when possible.

Undefined are usually things that haven't been declared and null is assigned in the code.

NaN stands for not a number. It is the result when methods or operations that can only be used on numbers has a value in it that is not a number

Always false falsys: empty string literals('',"",``),0,-0,NaN,false,null,undefined

Bitwise has uses, but typically are not used except in resource heavy programs that require good performance, mostly games. The only time I've ever seen them used was to replace some code that was causing a performance bottleneck in the main game loop

Chapter 3 - Arrays, Logic, and Loops

Not much new in this chapter for me. It is always best to use the built in methods to do things like sort and reverse. The algorithms for them have been coded to use the best Big O version possible which always helps performance

Sets are like arrays except they do not allow duplicate values to exist within it and are not indexed.

Weak sets and weak maps help with the memory leak issues caused by an object being in one being deleted and not being garbage collected. Weak sets and maps garbage collect any reference to the object preventing the leak

Chapter 4 - Functions

callbacks are used by passing functions as an argument to a function then use it within the function.

array iterators use callbacks to add some useful utilities to arrays.
