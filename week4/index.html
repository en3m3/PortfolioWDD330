Chapter 12 - Object Oriented Programming

encapsulation allows the internal workings of objects to be hidden leaving specific interfacing methods to interact with the objects

polymorphism allows processes to be used by multiple objects with the ability to be overridden for more specific implementation

inheritance allows objects to extend another object bringing with it all of it's methods and properties. The inherited properties and methods can be modified or overriden as well as new ones being added

javascript uses prototypes for creating classes. ES6 introduced classes but they still use prototyping in the background

classes are similar to inheritance except the class itself is not an object and an instance of it must be created to use it

constructor functions are called when a new instance of an object is created.

built-in constuctors: Object, Array, Function, String, Number, Boolean, RegExp, Date

can use literals instead of new for the built-in constructors

capitalize class names

classes are preferred over constructor statements

static methods are called from the class not an instance of the class

prototype is live. If new properties or methods are added all instances of it will also gain them

starting a variable with an underscore denotes a private property of a class. Getter and setter methods should be used to prevent access outside of the object to the private properties.

defineProperty() allows you to set the property discriptors

create allows to make a copy of an object

there are various ways to keep the this in a nested function, setting this to another variable such as that, self, _this, use bind(this), instead of a foreach loop use for-of pattern, use an arrow function

you can 'borrow' functions by setting them to a variable then using call()

Chapter 15 - Modern Javascript

libraries help reduce repetition by creating abstractions that make performing the actions easier

libraries have pros and cons. Adds utility most of the time but they can get large and increase difficulty due to needing to learn how to use it

modules are self contained code that can be easily be reused and only the api needs to be understood to useit

Node.js supported modules prior to ES6

transpiling allows use of newest js releases but maintains sometimes not yet supported code
